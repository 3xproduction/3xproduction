# DEPRECATED — 2D карта склада

Эта папка **удалена из UI 2026-04-22**. Здесь была карта склада в стиле
Miro/Figma: 4000×3000 виртуальный холст с секциями, которые можно было
перетаскивать, вращать, масштабировать (`react-rnd` + `@use-gesture/react`).

Заменена на **каталог-навигацию** (3 уровня):

```
/cells                                     — CellsIndex  (3 плитки подскладов)
/cells/:warehouseId/type/:type             — CellsTypeView (карточки секций)
/cells/:warehouseId/section/:sectionId     — CellsSectionView (реалистичный рендер)
```

Новый UI лежит в `../cells/` и `../sections/` (реалистичные рендеры
`ShelfSection`/`HangerSection`/`PlaceSection` перенесены из этой папки).

## Что осталось здесь

- `MapCanvas.jsx` — холст с зумом/паном/pinch/rotation (react-rnd + use-gesture).
- `ShelfSection.jsx`, `HangerSection.jsx`, `PlaceSection.jsx` — старые версии секций
  (актуальные копии — в `../sections/`).
- `autoLayout.js` — масонная автораскладка секций в сетку 3 колонки.

## Почему не удаляем

По просьбе пользователя — оставлено как архив: если каталог-подход
окажется неудачным или появится задача «вернуть карту», код под рукой.
Файлы никуда не импортируются, на бандл влияния нет
(tree-shaker Vite просто их не включает).

## БД

Поля `x_pos`, `y_pos`, `width`, `height`, `rotation` в `warehouse_sections`
(миграции 039, 051) **не удалены**. Новый UI их не пишет, но и не трогает —
останутся валидные значения у ранее размещённых секций. Это позволяет в любой
момент подключить обратно старый `MapCanvas` без миграции.

## Пакеты

`react-rnd` и `@use-gesture/react` остаются в `frontend/package.json` —
пока файлы этой папки их импортируют. Если решено окончательно удалить
карту — сносить папку целиком вместе с пакетами.
